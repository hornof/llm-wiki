## Definition
Karpathy's three-generation framework for how software is written:

- **Software 1.0**: Explicit rules. A programmer writes code that specifies every step.
- **Software 2.0**: Learned weights. A programmer specifies behavior by curating datasets and training neural networks. The programming artifact is the dataset and loss function, not code.
- **Software 3.0**: Prompting. LLMs trained on massive multitask corpora become programmable computers. The programming paradigm is the context window — "what is the piece of text to copy paste to your agent?"

## Why It Matters
The paradigm shift changes what engineers actually do. In Software 3.0, the engineer is no longer the person who writes the code — they are the person who specifies intent to an intelligent interpreter. This opens up automation of tasks that aren't even programs in the traditional sense (e.g., building a knowledge base from unstructured documents).

> "It's not just about programming and programming becoming faster. This is more general information processing that is automatable now." — Andrej Karpathy, AI Ascent 2026

> "What is the piece of text to copy paste to your agent? That's the programming paradigm." — Andrej Karpathy, AI Ascent 2026

## Current State
Actively shaping how practitioners think about product design and system architecture. The key implication is that many apps "shouldn't exist" — they were built in a Software 1.0/2.0 mindset when the same outcome can be achieved by passing raw inputs directly to an LLM.

**Canonical example (Karpathy)**: Building MenuGen — a full-stack app to OCR a restaurant menu and generate food images — then discovering the Software 3.0 version is a single prompt to Gemini with NanoBANANA overlay. The app was spurious; the paradigm made it obsolete.

**OpenClaw example**: The installer for OpenClaw is not a shell script — it is a block of text to paste to your agent. The agent reads the environment intelligently and debugs in a loop, which is more powerful than any shell script targeting multiple platforms.

## Implications for Founders and Builders
1. Don't ask "how do I speed up what exists?" — ask "what new things are now possible that weren't before?"
2. Existing apps may be spurious (built in the old paradigm) if the same output can be achieved with a raw prompt.
3. Infrastructure needs to become agent-native — docs written for humans, UIs designed for humans, and deployment flows requiring human clicks are all friction in the Software 3.0 world.

## Pre-history (Karpathy 2022)
Karpathy effectively states the Software 3.0 thesis in his 2022 essay *Deep Neural Nets: 33 years ago and 33 years from now*: "you will ask a 10,000,000X-sized neural net megabrain to perform some task by speaking... to it in English." The 2022 framing places the natural-language-to-mega-model interface as the endpoint of a 33-year scaling curve — four years before he names this paradigm Software 3.0 at AI Ascent 2026. The continuity between 2022 and 2026 is exactly the "macro-architecture is constant; scale is everything" thesis from the same essay. — [[karpathy-lecun1989-33-years]]
