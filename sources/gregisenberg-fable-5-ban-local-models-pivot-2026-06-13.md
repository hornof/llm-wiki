### 8-step local-models on-ramp guide

**First wiki-captured 8-step canonical local-models on-ramp** (Greg Isenberg framing):

| # | Step | Detail |
|---|---|---|
| 1 | **Start with the runtime** | Ollama or LM Studio — *"actually runs models on your machine"* |
| 2 | **Match model to hardware** | 7B = any laptop / 32B = good Mac 32GB+ / 70B = DGX Spark or maxed-out Mac Studio |
| 3 | **Know which model for which job** | Qwen 3 = all-around best / DeepSeek = reasoning + coding / Gemma 4 = phone-tier / Llama = biggest community + most fine-tunes |
| 4 | **Quantization** | Q4 or Q5 — *"how a model that 'needs' a server runs on your laptop. Learning this one concept changes everything."* |
| 5 | **Connect to agent** | Point Hermes or your agent stack at a local model |
| 6 | **Context window is your real constraint locally** | *"Cloud models give you huge context for free. Local models make you pay for it in memory."* |
| 7 | **Give local models tools** | *"A smaller local model with web search, file access, and code execution beats a giant model with none. The capability gap closes fast when you wire up the right tools. The model is the engine but the tools are the wheels."* |
| 8 | **Fine-tuning is more accessible than you think** | Day-1 not required but exists as upgrade-path |

**First wiki-captured operator-side canonical 4-model-recommendation taxonomy**: Qwen 3 (all-around) + DeepSeek (reasoning/coding) + Gemma 4 (tiny/phone) + Llama (community/fine-tunes).
