### Comment-thread structurally MAJOR — Alien's learning-loop-physical-location-test framing

**Alien @alienorg** (Jun 15) replies with **structurally-deepest reformulation of Nadella model-substitutability test**:

> *"the veteran-expertise-stays test is really a question about where the learning loop physically lives — if it sits inside a provider's stack, swapping the model is easy and leaving isn't"*

**First wiki-captured "learning-loop-physical-location-test" canonical framing** — reformulates Nadella's *"switch out a 'generalist' model without losing the 'company veteran' expertise"* test as a question about **where the learning loop physically lives**:

- **Learning loop inside provider's stack** → model-swap is easy, but vendor-lock-in via the loop infrastructure
- **Learning loop inside operator's substrate** → true vendor-portability of the *expertise* (not just the model)

**Implication**: Nadella's frontier-ecosystem thesis becomes operationally meaningful only when the learning-loop infrastructure sits on the operator's substrate, not the provider's. **First wiki-captured deep-structural critique that "frontier-ecosystem thesis = model-substitutability claim" depends on operator-side learning-loop-substrate-control**.
