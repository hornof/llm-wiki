**2. 100-decisions canonical-threshold framing**:
> *"The first instinct is to update the prompt after every action. Don't. One decision is noise. A single reject tells you almost nothing about what the user actually wants."*

**5 canonical-noise-sources articulation**:
- User was tired or distracted
- Input was weird in unrelated way
- Misclicked
- Moved fast on obvious case
- Testing the system on purpose

> *"100 decisions is signal. You can see a pattern: the user keeps rejecting candidates without startup experience. That's not a mood, that's a preference. Now you update."*

**First wiki-captured 100-decisions canonical-threshold for prompt-self-update**. **First wiki-captured 5-canonical-noise-sources articulation for canonical-single-decision-as-noise framing**.
