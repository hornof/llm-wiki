## Key Claims / Takeaways

**Canonical-thesis-opener** (Cherny canonical-direct-voice):

- *"As engineering, product, design, DS, etc. melt into a new kind of role, I was reflecting on what roles might look like in the future."*

**Canonical-5-archetype canonical-roster** (Claude Code team observation):

1. **Prototyper** — comes up with brand new ideas; churns out many ideas, most of which don't ship
2. **Builder** — quickly turns a prototype/idea into production-grade product/infra
3. **Sweeper** — cleans up the UI, simplifies the code and system, unships, optimizes performance
4. **Grower** — takes a product that has been built and iterates on it to improve Product-Market Fit
5. **Maintainer** — owns a mature system to make it secure, reliable, fast, and efficient as it scales

**Canonical-archetypes-not-tied-to-job-function canonical-articulation**:

- *"these roles are not really tied to job function — eg. across Anthropic, some designers match category 1, some 2, some 3; same for engineers, PM, DS."*
- **First wiki-captured canonical-archetype-vs-job-function canonical-distinction at canonical-Anthropic-organizational-tier**.

**Canonical-team-composition-by-product-stage canonical-articulation**:

- **Pre-PMF product**: needs people strong at **1+2+3** (Prototyper + Builder + Sweeper)
- **Growing post-PMF product**: needs **2+3+4** and some **5** (Builder + Sweeper + Grower + some Maintainer)
- **Strong PMF product**: needs **3+4+5** and some **2** (Sweeper + Grower + Maintainer + some Builder)
